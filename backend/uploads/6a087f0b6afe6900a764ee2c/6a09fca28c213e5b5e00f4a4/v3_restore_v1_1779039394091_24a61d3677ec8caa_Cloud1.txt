Hosting a website in public subnet and the database in private subnet
1) Create a VPC
Go to VPC → Create VPC
Select VPC only
Name: MyVPC
CIDR: 10.0.0.0/16
Click Create

Create Subnets
Public Subnet
Name: PublicSubnet
CIDR: 10.0.1.0/24
Private Subnet
Name: PrivateSubnet
CIDR: 10.0.2.0/24

Internet Gateway (for website)
Create Internet Gateway
Attach to VPC

Elastic IP:
allocate elastic ip and click on allocate

NAT Gateway:
Name: MyNAT
Availability: Zonal
Subnet: PublicSubnet
Connectivity: Public
Elastic IP: select allocated IP
Create

Route Tables
Public Route Table
Create PublicRT
Add route:
0.0.0.0/0 → Internet Gateway
Associate with PublicSubnet

Private Route Table
Create PrivateRT
Add route:
0.0.0.0/0->NAT Gateway
Associate with PrivateSubnet

2)Create and launch ec2 instance
Name: WebServer
Keep other options same and change Network settings:
VPC: MyVPC
Subnet: Public subnet
Auto-assign public IP: ✅ Enable
Download the key pair
Create a Security Group
Name:EC2-SG
SSH (22) → 0.0.0.0/0
HTTP (80) → 0.0.0.0/0
Launch Instance


3) Connect to the ec2 instance and execute
sudo yum update -y
sudo yum install httpd -y
sudo systemctl start httpd
sudo systemctl enable httpd

4) Go to amazon ec2 and create another instance
Name: DBServer
Keep other options same and change Network settings:
VPC: MyVPC
Subnet: Private subnet
Auto-assign public IP: Disable
Use existing key pair
Create a Security Group
Name:DB-SG
SSH (22) → Custom(EC2-SG)
MYSQL → Custom(EC2-SG)
Launch Instance

5)After connecting to ec2 instance of webserver, execute
nano MyKeyPair.pem
Open the pem file you have downloaded in notepad, copy the content and paste it in the file. 
To save press Ctrl+X, then press y and then press enter
Then execute,
chmod 400 MyKeyPair.pem
ssh -i MyKeyPair.pem ec2-user@private-ip-dbserver
Then execute,
sudo yum update -y
sudo yum install mariadb105-server -y
sudo systemctl start mariadb
sudo systemctl enable mariadb

6)Configure sql
sudo mysql
ALTER USER 'root'@'localhost' IDENTIFIED BY 'root123';
CREATE USER 'root'@'%' IDENTIFIED BY 'root123';
GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' WITH GRANT OPTION;
FLUSH PRIVILEGES;
EXIT;

root123 is the password for the database


7)sudo nano /etc/my.cnf.d/mariadb-server.cnf
Add:
[mysqld]
bind-address=0.0.0.0
and save
sudo systemctl restart mariadb

8)In the webserver
sudo yum install mariadb105 -y
mysql -h private-ip-dbserver -u root -p
enter the password as root123
now you will be able to use the database in public ec2 instance
